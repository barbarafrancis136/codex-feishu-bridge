# codex-feishu-bridge 使用说明

## 1. 这个插件是什么

`codex-feishu-bridge` 是一个本地运行的飞书/Lark 机器人桥接插件。

它把飞书消息转发给本机 Codex app-server，再把 Codex 的回复发回飞书。

```text
飞书 / Lark
  -> codex-feishu-bridge
  -> 本机 Codex app-server
  -> codex-feishu-bridge
  -> 飞书 / Lark
```

它适合这些场景：

- 想在手机飞书里继续使用电脑上的 Codex。
- 想把飞书群或私聊变成 Codex 的远程入口。
- 想让一个飞书会话绑定一个本地项目目录。
- 想在飞书里切换 Codex 线程、模型和推理强度。
- 想用飞书卡片审批 Codex 的本地操作请求。
- 想把飞书图片交给 Codex 看，并把 Codex 生成或找到的图片发回飞书。

## 2. 它能办哪些事

### 2.1 普通对话

你可以在绑定项目后直接发普通消息，插件会把消息转给 Codex。

Codex 回复会以飞书卡片显示。默认支持流式更新，也就是 Codex 一边生成，飞书卡片一边刷新。

### 2.2 绑定本地项目

一个飞书会话可以绑定一个或多个本地项目目录。

绑定后，普通消息会在对应项目目录里启动或继续 Codex 线程。

常用命令：

```text
/codex bind /absolute/path
/codex where
/codex workspace
/codex remove /absolute/path
```

### 2.3 管理 Codex 线程

插件可以读取 Codex runtime 可见的线程列表，并在飞书里切换。

常用命令：

```text
/codex new
/codex switch <threadId>
/codex message
```

能做到：

- 新建当前项目下的 Codex 线程。
- 切换到指定历史线程。
- 查看最近几轮对话摘要。
- 在同一个飞书会话里延续当前 Codex 线程。

### 2.4 控制模型和推理强度

插件启动时会读取 Codex 可用模型列表，并校验默认模型配置。

常用命令：

```text
/codex model
/codex model update
/codex model <modelId>
/codex effort
/codex effort <low|medium|high|xhigh>
```

能做到：

- 查看当前可用模型。
- 刷新模型列表。
- 为当前项目设置模型覆盖。
- 为当前项目设置推理强度覆盖。

### 2.5 停止正在运行的任务

如果当前线程正在执行，飞书端可以发送：

```text
/codex stop
```

插件会把停止请求转给 Codex runtime。

### 2.6 审批 Codex 操作

当 Codex 发起需要用户确认的操作时，插件会在飞书里发送审批卡片。

常用命令：

```text
/codex approve
/codex approve workspace
/codex reject
```

能做到：

- 批准本次操作。
- 批准当前 workspace 范围内的同类操作。
- 拒绝操作。

### 2.7 图片和附件

绑定项目后，可以直接给机器人发送图片。插件会把飞书图片下载到本机私有缓存，并作为 Codex 原生 `localImage` 输入进入当前轮。

也可以发送项目内文件：

```text
/codex send <relative-file-path>
```

例如：

```text
/codex send README.md
```

插件会检查路径在当前绑定项目内，再通过飞书接口发送。图片会走图片消息，`.opus/.mp4` 会走音频消息，其他文件会走普通文件消息。

Codex 也可以在回复里包含独立一行隐藏指令来回传当前项目内的文件：

```text
[[codex-feishu-send:relative/path/from/workspace]]
```

桥会上传该文件并从展示文本中移除这行指令。

## 3. 它不包含哪些东西

公开版只保留 Codex 和飞书连接能力。

不包含：

- 私有知识库。
- 私人任务系统。
- 记忆编译或召回脚本。
- 每日沉淀、日报、复盘系统。
- 特定团队的项目中枢。
- 其他本地自动化平台集成。
- 任何密钥、token、私有 ID、本地日志或个人工作区数据。

如果你需要这些能力，可以在这个插件外面单独做扩展，不建议混进核心桥接层。

## 4. 安装前准备

需要：

- Node.js 18 或更高版本。
- 本机可运行 Codex，并支持 `codex app-server`。
- 一个飞书/Lark 自建应用。
- 飞书应用开启机器人能力。
- 飞书应用开启长连接或事件订阅所需配置。

## 5. 安装

从 npm 安装：

```sh
npm install -g codex-feishu-bridge
```

或者在源码目录本地运行：

```sh
npm install
npm run feishu-bot
```

启动：

```sh
codex-im feishu-bot
```

## 6. 配置

插件会按顺序读取：

1. 当前目录的 `.env`
2. `~/.codex-im/.env`
3. 当前 shell 环境变量

最小配置：

```text
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxx
CODEX_IM_DEFAULT_CODEX_MODEL=gpt-5.3-codex
CODEX_IM_DEFAULT_CODEX_EFFORT=medium
CODEX_IM_DEFAULT_CODEX_ACCESS_MODE=default
```

常用可选配置：

```text
CODEX_IM_DEFAULT_WORKSPACE_ID=default
CODEX_IM_FEISHU_STREAMING_OUTPUT=true
CODEX_IM_FEISHU_CARDKIT_STREAMING=true
CODEX_IM_FEISHU_RETRY_MAX_ATTEMPTS=3
CODEX_IM_FEISHU_RETRY_BASE_DELAY_MS=300
CODEX_IM_CODEX_RPC_TIMEOUT_MS=45000
CODEX_IM_CODEX_TURN_START_TIMEOUT_MS=60000
CODEX_IM_STALE_TURN_TIMEOUT_MS=1800000
CODEX_IM_WORKSPACE_ALLOWLIST=/absolute/project-a,/absolute/project-b
CODEX_IM_CODEX_ENDPOINT=
CODEX_IM_SESSIONS_FILE=/Users/your-name/.codex-im/sessions.json
CODEX_IM_ATTACHMENTS_DIR=/Users/your-name/.codex-feishu-bridge/attachments
CODEX_IM_MAX_IMAGE_BYTES=10485760
CODEX_IM_MAX_ATTACHMENT_BYTES=104857600
```

说明：

- `CODEX_IM_WORKSPACE_ALLOWLIST` 为空时允许绑定任意绝对路径。
- `CODEX_IM_CODEX_ENDPOINT` 为空时，插件会尝试启动本地 `codex app-server`。
- `CODEX_IM_SESSIONS_FILE` 用于保存飞书会话和 Codex workspace/thread 的绑定关系。
- `CODEX_IM_ATTACHMENTS_DIR` 用于保存飞书图片和文件的本机私有缓存。

## 7. 飞书应用配置

事件订阅：

| 事件 | 标识 |
| --- | --- |
| 接收消息 | `im.message.receive_v1` |
| 卡片回传交互 | `card.action.trigger` |

推荐权限：

| 权限 | 标识 |
| --- | --- |
| 创建与更新卡片 | `cardkit:card:write` |
| 获取卡片信息 | `cardkit:card:read` |
| 以应用身份发消息 | `im:message:send_as_bot` |
| 读取用户发给机器人的单聊消息 | `im:message.p2p_msg:readonly` |
| 发送/删除表情回复 | `im:message.reactions:write_only` |
| 获取与上传图片或文件资源 | `im:resource` |

## 8. 第一次使用流程

1. 启动插件：

```sh
codex-im feishu-bot
```

2. 在飞书里给机器人发送：

```text
/codex bind /absolute/path/to/project
```

3. 查看绑定状态：

```text
/codex where
```

4. 直接发送普通问题或任务。

5. 需要新线程时：

```text
/codex new
```

6. 需要停止当前任务时：

```text
/codex stop
```

## 10. 常见问题

### 10.1 启动时报模型不合法

插件启动时会向 Codex 查询模型列表。如果 `CODEX_IM_DEFAULT_CODEX_MODEL` 不在列表里，会启动失败。

处理方式：

1. 先确认本机 Codex 可正常启动。
2. 换成当前 Codex 可用模型。
3. 重新启动插件。

### 10.2 飞书能收到消息，但 Codex 没回复

可能原因：

- Codex app-server 没启动成功。
- 本机网络或权限拦截了 WebSocket。
- 当前项目路径不可访问。
- 线程仍在运行，飞书端等待上一次任务完成。

可以尝试：

```text
/codex where
/codex stop
/codex new
```

### 10.3 无法绑定项目

如果设置了 `CODEX_IM_WORKSPACE_ALLOWLIST`，只能绑定白名单里的路径。

检查：

- 路径必须是绝对路径。
- 路径必须存在。
- 当前运行插件的系统用户必须有权限读取该目录。

### 10.4 文件发送失败

`/codex send` 只能发送当前绑定项目里的相对路径文件。

检查：

- 不要传绝对路径。
- 文件必须在当前绑定项目目录内。
- 文件大小不能超过飞书接口限制。
- 飞书应用需要 `im:resource` 权限。

### 10.5 图片没有进入 Codex

检查：

- 飞书应用需要 `im:resource` 权限。
- 图片不能超过 `CODEX_IM_MAX_IMAGE_BYTES`。
- 运行插件的系统用户需要能写入 `CODEX_IM_ATTACHMENTS_DIR`。
- 当前 Codex app-server 需要支持 `localImage` 输入。
